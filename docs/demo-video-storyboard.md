# Demo Video Storyboard — EC Growth Studio AI（上司デモ動画用構成案）

> 上司・社内レビュー・初回商談向けに **画面録画ベースのデモ動画** を作るための構成案。
> 想定: ナレーションは別撮り or テロップ字幕、画面は実ブラウザのスクリーンキャプチャ。
> 関連: 話す台本は [demo-script.md](demo-script.md)、両プロダクト比較は [insight-studio-vs-ec-growth-studio.md](insight-studio-vs-ec-growth-studio.md)
>
> **重要**: 本動画は EC Growth Studio AI の世界観を伝える目的。Insight Studio の動画ではない。
> ただし **「Insight Studio で実際に動いている技術基盤がある」** ことは1カット必ず触れる。

---

## 0. 共通の準備（撮影前の必須チェック）

### 0-1. 環境

- 本番URL: <https://ec-growth-studio-ai.vercel.app>
- ローカル: `npm run dev` → <http://localhost:5173>
- ブラウザ: Chrome / Edge、**幅 1280px 以上**（lg ブレイクポイントが安定する）
- 拡張機能: 全てオフ（広告ブロッカー / Grammarly などのオーバーレイを消す）
- ブックマークバー / 開発者ツールは閉じる

### 0-2. データ

- サンプルCSV（`samples/csv/`）:
  - `orders_sample.csv`
  - `ga4_sample.csv`
  - `ads_sample.csv`
- 撮影直前に `/app/data-import` の **「サンプルデータで試す」** または **「取込済データを解除」** を押し、
  **クリーンな状態** から開始する。CSV-first の流れが映える

### 0-3. 安全確認（撮影中・編集中）

| 項目 | やること |
| --- | --- |
| 顧客実データ | **絶対に映さない**。`samples/csv/` の合成データのみ使用 |
| APIキー / トークン | 画面に出さない（DevTools / .env / ターミナルを映さない） |
| AI考察文言 | 「**AIが生成する想定の例文**」とテロップで必ず明示 |
| 「接続する」ボタン | デモ中に押さない（UIモックアップで実APIには繋がらない） |
| 競合製品 | スクリーンショットを画面共有しない |

### 0-4. 動画フォーマット推奨

- 解像度: 1920×1080（FullHD）
- フレームレート: 30fps
- 音声: 別撮りナレーション or AIナレーション + BGMフェード
- テロップ: 各カットに **キーメッセージ1行** + 必要なら **注記**（「AI実接続は Phase 4」など）
- カーソル軌跡: できれば誇張して表示（OBS のクリックエフェクト推奨）

## 4. 撮影時の注意（全動画共通）

### 4-1. やってはいけない

| NG | 理由 |
| --- | --- |
| 顧客の本物CSVを使う | **重大なプライバシー違反**。`samples/csv/` の合成データのみ |
| APIキー / トークンを映す | **重大なセキュリティ事故**。DevTools / .env / ターミナル / Settings画面を映さない |
| AI考察を「これがAIの出力」と断定する | 現状は静的サンプル。**「想定の例文」と必ずテロップで明示** |
| 「接続する」ボタンを押す | UIモックアップで実APIには繋がらない |
| Sidebar から未実装画面を開く | **NotFound に飛ぶ壊れナビは動画に映らない**よう、現状7画面（ダッシュボード / 売上要因分析 / データ取込 / AI考察 / 商品ページ / 施策ボード / 月次レポート）以外触らない |
| Insight Studio の画面を出す | 顧客実データが映る可能性。テロップ + ロゴのみで表現 |
| 競合製品の比較スクショ | 口頭・テロップで表現 |

### 4-2. 推奨

- 各シーンの最初に **「現MVPは静的データ + 一部CSV実データ」** を1テロップだけ入れる
- カーソルクリック時にエフェクト（OBS の Click Highlighter）
- BGM は控えめ（音量 -20dB 以下）、ナレーションが主役
- テロップフォントは **Manrope or 游ゴシック** で統一（Insight Studio とブランド系統を揃える）
- 配色: **Navy基調**（プロダクトのブランドカラー） + アクセントに emerald

### 7-2-2. localStorage / 取込状態（撮影直前に1回だけ）

撮影は **必ずクリーン状態から始める**。CSV-first の演出が映えるため:

1. ブラウザで <https://ec-growth-studio-ai.vercel.app/app/data-import> を開く
2. 画面下部の **「サンプルデータで試す」**（または各CSVの「取込を解除」）を押し、注文 / GA4 / 広告 すべての取込状態を解除する
3. ページをリロードして **取込済みバナーが消えている** ことを確認
4. ダッシュボード `/app` を開いて **「CSV取込済み」バナーが出ていない** ことを確認

> ⚠️ Production 環境を使う場合、別タブで実顧客データを取込んでいないか必ず確認する。
> サンプルCSV（`samples/csv/orders_sample.csv` `ga4_sample.csv` `ads_sample.csv`）以外は **絶対に映さない**。

### 7-2-3. CSV取込の順番（10分版・5分版の実演で固定）

3CSV を **必ずこの順番** で投入する（要因分解の実値ラベルが段階的に増える順）:

| 順 | ファイル | 直後に見せる画面 | 言うこと |
| --- | --- | --- | --- |
| 1 | `orders_sample.csv` | `/app/data-import` 取込結果 → `/app` の取込バナー | 「売上 / 注文 / AOV が実値で再計算」 |
| 2 | `ga4_sample.csv` | `/app/data-import` GA4取込結果 → `/app/revenue-analysis` の要因カード | 「セッション / CVR が GA4実値に切替」 |
| 3 | `ads_sample.csv` | `/app/data-import` 広告取込結果 → `/app/revenue-analysis` の下部 チャネル別効率 / 効率悪化キャンペーン | 「ROAS / CPC / 効率悪化キャンペーンが追加表示」 |

3分版の場合は **取込済み状態のスクリーンショット2〜3枚** を編集で差し込むだけにして、撮影中は取込操作を省略してもよい。

### 7-2-5. 録画NG事項（撮影中・編集中ともに守る）

| NG | 何が起こるか |
| --- | --- |
| 顧客の本物 CSV を取込む | プライバシー違反。**`samples/csv/` の合成データのみ使用** |
| `/app/data-import` の **API連携セクションの「Phase 3 で対応」「Phase 4 で対応」ボタン** をクリック | 何も起きない（disabled 仕様）。動画では押さず、Pill ラベルだけ拡大して見せる |
| 各画面の Topbar の **モックボタン**（例: 「月次レポート生成」「次に実行する施策を確認」「AI考察から施策生成」「施策ボードに反映」「PDFで出力」など） | 多くは onClick 未定義のため、画面遷移は発生するが新規データは生成されない。**Topbar の青ボタンは押すなら遷移用のみ**（"次に実行する施策を確認" → /app/action-board は OK） |
| Sidebar から未実装画面に飛ぶ | 現状は実装済み7画面のみだが、誤って **ブラウザのアドレスバーに直打ち** しない（`/app/foo` 等は NotFound） |
| ブラウザの **戻る / 進む** を多用 | 画面のスクロール位置が乱れて編集しづらくなる。基本は Sidebar からの遷移 |
| **DevTools / .env / ターミナル / Network タブ** を1秒でも映す | API キー漏洩のリスク。**録画範囲はブラウザのコンテンツ領域のみ** |
| Insight Studio の画面・実顧客名・実 GA4 プロパティID | 一切映さない。テロップ + ロゴでのみ言及 |
| 「これは本物のAIが動いています」と断定する | 現MVPは静的サンプル。**「AIが生成する想定の例文」と必ずナレーションで添える** |
